Why it matters

AI behavior changes are hard to root-cause because prompts, model settings, and tool responses drift over time. ReplayKit is designed to answer: "Why did this run behave differently?"

Without ReplayKit
With ReplayKit
  • Non-deterministic changes are hard to debug
  • Logs are incomplete or too sensitive to share
  • Reproductions depend on external services staying identical
  • It is difficult to isolate the first breaking change
  • Deterministic .rpk artifacts for stable replay
  • Offline replay (stub) plus selective rerun (hybrid)
  • Structured diff with first-divergence detection
  • Redacted bundle export plus integrity verification

What it demonstrates

Determinism engineering, provider-agnostic capture, security-first redaction defaults, and cross-platform CI validation.

System flow

ReplayKit captures boundary events and serializes them into deterministic artifacts that can replay offline.

  1. Capture model requests, prompts, tool calls, and outputs.
  2. Normalize and redact sensitive fields.
  3. Store run state as a deterministic `.rpk` bundle.
  4. Replay and diff runs to find first divergence.

Reliability layers

  • Deterministic serialization for reproducible state.
  • Provider-agnostic adapters for model interfaces.
  • CI coverage across Linux, macOS, and Windows.
  • Artifact checksums to verify integrity.

Quick workflow

Capture one run, replay it locally, then diff it against a modified run to pinpoint behavior changes.

replaykit capture --config examples/support_bot.yaml
replaykit replay --artifact runs/support_bot.rpk
replaykit diff --base runs/baseline.rpk --head runs/new_prompt.rpk

Expected output

  • Replay summary with deterministic step counts.
  • First-divergence report with exact event path.
  • Machine-readable diff output for CI gating.

Security model

ReplayKit is designed for least-privilege sharing of reproducible runs, even across teams.

  • Redaction defaults for API keys, secrets, and sensitive payloads.
  • Export controls for shareable, minimal artifacts.
  • Integrity verification before replay or review.

Operational safeguards

  • Offline replay mode to avoid accidental data egress.
  • Deterministic logs to prevent hidden mutation drift.
  • Clear diff provenance to support audit trails.

Artifact Explorer

Each `.rpk` package captures enough execution context to replay, inspect, and compare runs consistently.

manifest.json
Run metadata, adapters, schema version, and checksums.
events.ndjson
Ordered capture stream of prompts, tool IO, and model outputs.
redaction_map.json
Tokenized placeholders for sensitive fields, never raw secrets.
diff_snapshot.json
Optional precomputed indices for faster first-divergence lookup.
